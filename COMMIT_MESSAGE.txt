feat(ai): production-ready AI infrastructure foundation

Added core infrastructure for real LLM integration:

FEATURES:
- Multi-provider LLM configuration system (Groq, OpenAI, Anthropic, local)
- Comprehensive error handling hierarchy (AIError, RateLimitError, ValidationError, TokenLimitError, etc)
- Token counting utilities for prompt/response size estimation  
- Async utility library with retry logic, exponential backoff, and timeout handlers
- Structured logging system for AI operations with performance metrics

TYPE DEFINITIONS:
- src/types/ai/config.ts: AIConfig, ModelConfig, ProviderConfig interfaces

NEW IMPLEMENTATIONS:
- src/lib/utils/ai/asyncUtils.ts: Retry, timeout, batching, pooling utilities  
- src/services/ai/config/aiConfig.ts: Configuration manager with validation
- src/services/ai/errors/aiErrors.ts: Custom error classes for graceful handling
- src/services/ai/utils/tokenCounter.ts: Token estimation and management
- src/services/ai/observability/logger.ts: Structured logging with metrics

DEPENDENCIES ADDED:
- groq-sdk: Groq LLM API client
- @supabase/supabase-js: Database and authentication
- transformers: Local embedding models
- dotenv: Environment variable loading

CONFIGURATION:
- Updated .env.example with GROQ_API_KEY and Supabase credentials
- Added support for environment-based configuration
- Implemented validation for all config values

ARCHITECTURE DECISIONS:
- Type-safe config with Zod validation prevents runtime errors
- Error hierarchy enables graceful failure handling and recovery
- Token management prevents context window overflow and reduces costs
- Retry logic with exponential backoff handles transient failures
- Structured logging enables observability and debugging
- Modular provider support allows easy switching between LLMs

INTEGRATION POINTS:
- Agents can now call real LLMs using GroqProvider
- Token counting integrates with retrieval for context management
- Error handling works seamlessly with orchestrator pipeline
- Logging tracks performance metrics across entire system
