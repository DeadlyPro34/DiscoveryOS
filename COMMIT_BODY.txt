DATABASE SCHEMA
===============
Tables Created:
1. workspaces - Organization containers
2. workspace_members - Role-based access
3. projects - Project containers
4. documents - File uploads with tracking
5. chunks - Text chunks for RAG
6. embeddings - Vector embeddings
7. conversations - Chat sessions
8. messages - Message history
9. processing_logs - Audit trail
10. rag_metrics - Performance metrics

RAG PIPELINE STAGES
===================
1. UPLOADED → 2. PARSING → 3. NORMALIZING → 4. CHUNKING
→ 5. EMBEDDING → 6. INDEXING → 7. COMPLETED (or FAILED)

CHUNKING STRATEGIES
===================
1. FIXED_SIZE - Fixed chunks with configurable overlap
2. SEMANTIC - Sentence/paragraph-aware
3. PARAGRAPH - Paragraph-based splitting
4. SENTENCE - Sentence-based splitting
5. HIERARCHICAL - Multi-level combination
